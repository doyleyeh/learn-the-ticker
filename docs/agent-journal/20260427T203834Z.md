# T-119: Wire local frontend API access and backend CORS

## Scope

Fix the MVP local testing blockers found after T-118: frontend chat, export, and comparison paths could hit missing Next API routes or fallback fixtures while the FastAPI backend worked directly. Keep the change deterministic, local-first, and aligned with Frontend Design and Workflow v0.4.

## Changes

- Added a shared frontend API endpoint helper so chat, export, and comparison helpers prefer `NEXT_PUBLIC_API_BASE_URL` or `API_BASE_URL`, use `http://127.0.0.1:8000` in local development, and keep relative `/api` fallback for proxied deployments.
- Added a Next `/api/:path*` rewrite to the configured FastAPI backend for local development and deployments with API-base configuration.
- Added `build_cors_settings` and FastAPI `CORSMiddleware` wiring from `CORS_ALLOWED_ORIGINS`.
- Updated local env examples and Docker Compose to include both `localhost:3000` and `127.0.0.1:3000` local origins.
- Updated SPEC, EVALS, TASKS, the runtime gap audit, the local fresh-data runbook, the launch readiness matrix, and go/no-go checklist to reflect the T-119 baseline and T-120 through T-123 backlog.
