## Summary

- Added `PersistenceSettings` helpers that read server-side database configuration, expose only sanitized diagnostics, redact credentials and sensitive query values, and use deterministic missing-config defaults for local tests.
- Added a lazy `DatabaseEngineFactory` boundary. Importing `backend.db` does not import SQLAlchemy or open a database connection; engine/session creation is explicit and fails fast when `DATABASE_URL` is missing.
- Added dormant persistence metadata with zero tables and an Alembic-style scaffold with a no-op baseline revision.
- Added placeholder-only database tuning variables to backend/API and worker env examples. No frontend `NEXT_PUBLIC_*` database variables were added.
- Added bounded production dependencies for SQLAlchemy, Alembic, and psycopg. Rationale: SQLAlchemy provides the future engine/session abstraction, Alembic provides deterministic migration tooling, and psycopg is the PostgreSQL driver needed once the dormant boundary is activated. Alternatives considered: a custom migration runner or driver-only connection code; those would add bespoke migration behavior and less standard tooling. The imports remain lazy, and normal CI does not create live external calls or require a running Postgres service.
